# Чего не делать

## Не выдумывать факты о пользователе

Не вставлять имена, фамилии, профессии, биографические детали по догадке из email, имени системной папки или названия проекта. В первом же заказе на этот сайт я вставил «frontend-разработчик Михаил Горбачёв» — и имя (натянул из email), и профессию (взял из воздуха) пришлось потом переделывать. Если нужна личная информация, которой нет в текущем диалоге, в `index.html` или в памяти — спрашивать явно.

## Не править тексты биографии без запроса

«Обо мне», таймлайн опыта, описания этапов карьеры — это реальная биография пользователя, а не декоративный плейсхолдер. Вёрстку и стили можно менять, смыслы — нет.

## Не тянуть в проект сложные инструменты

Сборщики, фреймворки, TypeScript, линтеры, препроцессоры CSS, системы шаблонов — ничего из этого **без явной просьбы**. Стек намеренно минимальный, чтобы пользователь сам мог открыть любой файл и поправить руками.

## Не предлагать выносить тексты в отдельный файл

Разбирали этот вопрос: для одностраничного портфолио с одним автором выгоды нет, появляется только риск рассинхронизации. Если всерьёз понадобится — это когда будет i18n, CMS, A/B-тесты или большая команда редакторов.

## Не использовать PAT для пуша

SSH уже настроен. Не предлагать «создайте токен и введите в keychain» — это шаг назад.
