# Спецификация: рабочая форма обратной связи

Версия 1.0. Собрано в диалоге 2026-04-21. Описывает **целевое поведение** формы — на сегодня ещё не реализовано, на сайте стоит заглушка.

## Обзор

Посетитель заполняет форму на странице, JS отправляет данные на собственный бэкенд (endpoint на том же VPS). Бэкенд валидирует, рассылает уведомления (email + Telegram), отвечает автору автосообщением и возвращает фронту успех или ошибку.

## Что видит посетитель

1. Заполняет форму и нажимает «Отправить».
2. Кнопка показывает состояние «Отправка…» (блокируется на время запроса).
3. На успех — форма скрывается, вместо неё появляется блок:
   > **Спасибо!**
   > Сообщение отправилось. Отвечу в течение пары дней.
4. На ошибку (сервер недоступен / валидация упала) — показывается блок:
   > **Не удалось отправить.**
   > Попробуйте ещё раз или напишите на `neiro@unisender.com` напрямую.

## Поля формы

| Поле      | Тип      | Обязательное | Валидация                                        |
| --------- | -------- | ------------ | ------------------------------------------------ |
| Имя       | text     | да           | 1–100 символов                                   |
| Email     | email    | да           | корректный формат, до 200 символов               |
| Telegram  | text     | нет          | до 50 символов, можно с `@` или без, можно пусто |
| Сообщение | textarea | да           | 10–5000 символов                                 |

Валидация — и на клиенте (для быстрой обратной связи), и на сервере (потому что клиенту верить нельзя).

## Защита от спама — honeypot

В форму добавляется скрытое поле, которое реальный пользователь не видит:

- `<input name="website">`, визуально уведён за экран через `position: absolute; left: -9999px;`, не попадает в таб-навигацию (`tabindex="-1"`) и не заполняется автозаполнением (`autocomplete="off"`).

Боты заполняют это поле по привычке. Если `website` на сервере не пустое — сервер **тихо** возвращает успех и ничего не отправляет. Бот думает, что прошло, вы ничего не получаете.

CAPTCHA пока не ставим. Если honeypot пропустит заметный поток спама — добавим Cloudflare Turnstile (бесплатный, без «кликай на пожарные гидранты»).

## Куда улетает сообщение

При успешной отправке происходит **три** действия параллельно.

### 1. Email автору на `shymovskaia73@gmail.com`

- **Тема:** `Сообщение с портфолио: <имя>`
- **Тело (plain text):**
  ```
  Имя:      <имя>
  Email:    <email>
  Telegram: <telegram или «—»>

  <сообщение>

  ———
  IP: <ip>
  UA: <user-agent>
  ```

Technical: через SMTP. Проще всего — Gmail SMTP с App Password от этого же ящика.

### 2. Telegram в ЛС `@mikeshumovski`

- **Текст сообщения:**
  ```
  📮 Новое сообщение с портфолио

  Имя: <имя>
  Email: <email>
  Telegram: <telegram или —>

  <сообщение>
  ```

Technical: Telegram Bot API. Нужен бот (создаётся через `@BotFather`) и `chat_id` (получается после того, как пользователь сам напишет своему боту).

### 3. Автоответ отправителю на email из формы

- **Тема:** `Спасибо, сообщение получено`
- **Тело:**
  ```
  <имя>, спасибо! Сообщение получено, отвечу в течение пары дней.

  — Михаил Шумовский
  ```

Нужно, чтобы отправитель был уверен, что форма сработала (на случай, если страница закрылась до окна «Спасибо»).

## Техническая сторона (высокоуровнево)

- **VPS:** отдаёт статику сайта + крутит маленький Node-сервер.
- **Бэкенд:** одна ручка `POST /api/contact`, принимает JSON, возвращает JSON. Библиотеки: `express`, `nodemailer`, `express-rate-limit`.
- **Nginx:** перед Node. Отдаёт статику, проксирует `/api/*` на `localhost:<порт>`.
- **HTTPS:** Let's Encrypt через Certbot. Без HTTPS современные браузеры будут ломать fetch из-за mixed content.
- **Rate limit:** 5 отправок в 15 минут с одного IP. Чтобы случайный «F5 спамер» не залил ящик.
- **Процесс-менеджер:** PM2 или systemd, чтобы Node-сервер переживал перезагрузку VPS.
- **Секреты:** `.env` на сервере (SMTP-пароль, токен бота, chat_id). В git не коммитим, `.env` добавится в `.gitignore`.

## Что нужно подготовить (чек-лист перед стартом работ)

- [ ] Домен + VPS (и свежий Ubuntu/Debian).
- [ ] Включить 2FA на `shymovskaia73@gmail.com` и сгенерировать **App Password** для SMTP (обычный пароль от почты Gmail для SMTP больше не принимает).
- [ ] Создать Telegram-бота через `@BotFather`, сохранить токен.
- [ ] Написать своему боту хоть одно сообщение (например, `/start`) и вытащить `chat_id` — без этого бот не сможет инициировать переписку.
