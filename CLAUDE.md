# Портфолио Михаила Шумовского

Одностраничный сайт-визитка. Пользователь — **Михаил Шумовский**, главный редактор «Нейромедиа» (проект Unisender), НЕ разработчик. Темы, в которых он работает: прикладные нейросети (промпты, сравнение моделей, детект ИИ-текста) и цифровой маркетинг (email, CRM, SEO, аналитика).

## Стек

Чистый HTML/CSS/JS без сборки и зависимостей. Не предлагать React/Next/Tailwind и т. п. — проект намеренно минимальный, чтобы пользователь мог сам править файлы.

## Файлы

- `index.html` — разметка всех секций
- `styles.css` — оформление, тёмная тема с градиентами, CSS-переменные в `:root`
- `script.js` — обработка формы (показ «спасибо»), год в футере, hover-подсветка карточек
- `.gitignore` — `.DS_Store`, `node_modules/`, логи, папки IDE

## Секции сайта (в порядке появления)

1. **Hero** (`<header class="hero">`) — имя, роль, CTA на Telegram и форму
2. **Обо мне** (`#about`) — два абзаца о карьерном пути и инфостиле
3. **Опыт** (`#experience`) — таймлайн из 4 этапов: Нейромедиа → Конверт → автор Unisender → фриланс
4. **О чём пишу** (`#topics`) — две колонки тем (нейросети / маркетинг)
5. **Избранные материалы** (`#works`) — 3 карточки-ссылки на реальные статьи + кнопки на профили
6. **Где публикуюсь** (`#publications`) — сетка логотипов площадок (Unisender, vc.ru, Хабр, TexTerra, SendPulse, SMMplanner, Ак Барс банк, Sendsay). Логотипы — через `https://www.google.com/s2/favicons?domain=X&sz=128`, ключи не нужны.
7. **Канал** (`#channel`) — большая карточка с Telegram-иконкой (инлайн SVG)
8. **Связаться** (`#contact`) — плашка с email редакции + форма (имя/email/сообщение)

## Как запускать

Лучше через локальный сервер с live-reload:

```
npx --yes live-server --port=5173 --no-browser --quiet
open http://localhost:5173
```

Либо просто открыть `index.html` двойным кликом (тогда без автообновления).

## Форма обратной связи

Сейчас чисто клиентская: на `submit` скрывает форму и показывает блок «Спасибо». Реальной отправки нет. Если пользователь попросит «сохранять сообщения» — подключать бэкенд или сторонний сервис (Formspree, Web3Forms и т. п.), обсуждать варианты, а не выбирать молча.

- Не добавлять сборщики, фреймворки, TypeScript, линтеры без явной просьбы.
