Одностраничный сайт-визитка. Пользователь — **Михаил Шумовский**, главный редактор «Нейромедиа» (проект Unisender), НЕ разработчик. Темы, в которых он работает: прикладные нейросети (промпты, сравнение моделей, детект ИИ-текста) и цифровой маркетинг (email, CRM, SEO, аналитика).

## Стек

Чистый HTML/CSS/JS без сборки и зависимостей. Не предлагать React/Next/Tailwind и т. п. — проект намеренно минимальный, чтобы пользователь мог сам править файлы. Структура: один `index.html` + стили в `css/` по компонентам + скрипты в `js/` по задачам.

## Файлы

- `index.html` — единственный HTML-файл (без сборки разбивать не стали), в `<head>` подключены 12 `<link>`, в конце `<body>` — 3 `<script>` в порядке: `footer-year.js`, `contact-form.js`, `card-hover.js`.
- `css/` — стили, один файл = одна зона ответственности:
  - `tokens.css` — только `:root` переменные. Менять цвета/радиусы/размеры — сюда.
  - `base.css` — reset, typography (h1–h3, p), `.container`, `.container-narrow`, стили inline-ссылок в тексте (`.tagline a`, `.timeline-content a`).
  - `buttons.css` — `.btn` + варианты (`primary`, `ghost`, `wide`).
  - `sections.css` — каркас `.section`, `.section-alt`, `.section-intro`, футер.
  - `hero.css` — шапка: фон с градиентами, сетка, `.eyebrow`, `.tagline`, `.hero-actions`.
  - `about.css` — двухколоночная сетка «Обо мне».
  - `timeline.css` — таймлайн опыта (точки, вертикальная линия, шапки этапов).
  - `topics.css` — блок «О чём пишу» (две карточки со списками).
  - `cards.css` — карточки избранных материалов (`.projects-grid`, `.card`, hover-подсветка через `--mx/--my`, теги, `.works-footer`).
  - `logos.css` — сетка логотипов площадок.
  - `channel.css` — Telegram-карточка.
  - `form.css` — форма обратной связи, блок «Спасибо», плашка с email редакции.
- `js/` — скрипты, каждый обёрнут в IIFE и защищён от отсутствия нужных элементов:
  - `footer-year.js` — подставляет текущий год в `#year`.
  - `contact-form.js` — обработчик submit формы → скрывает её и показывает блок «Спасибо».
  - `card-hover.js` — обновляет CSS-переменные `--mx`/`--my` на карточках по движению курсора (для спотлайт-подсветки из `cards.css`).
- `.gitignore` — `.DS_Store`, `node_modules/`, логи, папки IDE.

## Секции сайта (в порядке появления)

1. **Hero** (`<header class="hero">`) — имя, роль, CTA на Telegram и форму
2. **Обо мне** (`#about`) — два абзаца о карьерном пути и инфостиле
3. **Опыт** (`#experience`) — таймлайн из 4 этапов: Нейромедиа → Конверт → автор Unisender → фриланс
4. **О чём пишу** (`#topics`) — две колонки тем (нейросети / маркетинг)
5. **Избранные материалы** (`#works`) — 3 карточки-ссылки на реальные статьи + кнопки на профили
6. **Где публикуюсь** (`#publications`) — сетка логотипов площадок (Unisender, vc.ru, Хабр, TexTerra, SendPulse, SMMplanner, Ак Барс банк, Sendsay). Логотипы — через `https://www.google.com/s2/favicons?domain=X&sz=128`, ключи не нужны.
7. **Канал** (`#channel`) — большая карточка с Telegram-иконкой (инлайн SVG)
8. **Связаться** (`#contact`) — плашка с email редакции + форма (имя/email/сообщение)

## Как запускать

Лучше через локальный сервер с live-reload:

```
npx --yes live-server --port=5173 --no-browser --quiet
open http://localhost:5173
```

Либо просто открыть `index.html` двойным кликом (тогда без автообновления).

## Форма обратной связи

Сейчас чисто клиентская: на `submit` скрывает форму и показывает блок «Спасибо». Реальной отправки нет. Если пользователь попросит «сохранять сообщения» — подключать бэкенд или сторонний сервис (Formspree, Web3Forms и т. п.), обсуждать варианты, а не выбирать молча.
